## What this project is

A personal productivity system for a product manager. Trello is the source of truth for tasks. Claude Code syncs those tasks to local markdown files, which can be read in Obsidian or any editor.

## Project structure

```
Tasks/
  candidates/candidates.md     — Cards from "Candidates for the week" list
  in-progress/in-progress.md  — Cards from "Doing" and "Waiting" lists (split into ## sections)
  radar/radar.md               — Cards from "Key areas of attention" list
  done/done.md                 — Cards from "Done" list
  goals/goals.md               — Long-term goals from Trello (no checkboxes; context only)

Research/

  YYYY-WXX.md                  — Weekly research digest output
.claude/
  skills/                      — Skill definitions (one directory per skill, each with SKILL.md)
  settings.local.json          — Project settings (gitignored — not committed)
.mcp.json                      — Zapier and Notion MCP server config
[YourOrg]/
  context/         — Org-specific context (gitignored — stays local)
  [YourTeam]/
    Tasks/         — AI-ready task files generated by /task-write
```

- **Board ID**: visible in the Trello board URL — `trello.com/b/BOARD_ID/board-name`

  or use the Trello API sandbox at developer.atlassian.com

## File format conventions

Every task card uses this format:

```markdown
- [ ] **Card title** <!-- trello:CARD_ID -->
  `label1` · `label2` · Due: YYYY-MM-DD
  [Readable link text](url)
  > 1-2 sentence summary of what this task involves and what action to take
  - [x] Completed checklist item
  - [ ] Incomplete checklist item
```

- Trello card IDs are stored in HTML comments: `<!-- trello:ID -->`
- Goals have no checkboxes (they are context, not tasks)
- Labels use inline code spans: `` `Label` ``
- Omit metadata line if no labels and no due date
- Omit links line if no URLs in description
- Omit summary if description is empty
- Checklist items are display-only — checklist state is not synced back to Trello

## Sync conventions

- **Trello is authoritative** for all fetched lists — cards absent from Trello response are dropped from local files
- **Checked boxes `[x]`** in local files (or the weekly summary) trigger a move-to-Done on next sync
- **New items** without a `<!-- trello:ID -->` comment are created as new Trello cards
- Sync order: read local → fetch Trello → push local changes → write files (no re-fetch)
- Only rewrite a file if something actually changed (preserves timestamp)

## Notion Squad Board

The squad Notion database holds shared squad tasks (not personal Trello tasks). These live in a completely separate namespace — `/trello-sync` never touches `Tasks/squad/`.

**Database ID:** *(set this once you have it from the Notion URL: `notion.so/your-org/<database-id>?v=...`)*

| Item | Value |
|---|---|
| Local file | `Tasks/squad/squad.md` |
| Skill | `/notion-sync` |
| Source of truth | Notion |
| Sync trigger | Checked `[x]` boxes → status set to Done in Notion |

**Sync conventions:**
- Notion is authoritative — tasks absent from Notion (status = Done or deleted) are dropped from local file
- Checked `[x]` in `squad.md` → status pushed to Done on next `/notion-sync` run
- New `[ ]` items without a `<!-- notion:PAGE_ID -->` comment → created as new Notion pages
- `Tasks/squad/` is never read or written by `/trello-sync`

**MCP:** Notion MCP server is configured in `.mcp.json` (project-scoped, same as Zapier). Authenticate once via `/mcp` in Claude Code after opening the project.

## Skills

Skills live under `.claude/skills/` and are slash commands invoked in Claude Code.

| Skill | Command | Description |
|---|---|---|
| trello-sync | `/trello-sync` | Two-way sync between your Trello board and local files |
| trello-add | `/trello-add` | Quick-add a new card to your board |
| trello-update | `/trello-update` | Comment, move, or update description on an existing card |
| notion-sync | `/notion-sync` | Sync from a specific Notion board into `Tasks/squad/squad.md` |
| digest | `/digest` | Generate weekly research digest from RSS + web search |
| task-write | `/task-write` | Generate an AI-ready task file with full context for Claude Code |

## Git

After completing any task that modifies files — including skill runs (`/trello-sync`,
`/trello-add`, `/notion-sync`, `/digest`, `/task-write`) and direct edits to skills or config — do
the following automatically, without waiting to be asked:

1. Stage only the files that were changed.
2. Commit with a concise message describing what was done.
3. Push to `origin/main` immediately after committing.

Work directly on `main` — no feature branches needed for this repo.

## MCP

The Zapier MCP server provides Trello tools. Config is in `.mcp.json`. If tools are unavailable, re-authenticate via `/mcp` in Claude Code.

The Notion MCP server provides Notion tools. Config is in `.mcp.json`. If tools are unavailable, re-authenticate via `/mcp`.

## Self-improvement

After completing any skill run, evaluate whether the skill instructions were complete and accurate:

- If you had to improvise, work around unclear instructions, or the result was suboptimal in any way — propose a minimal, targeted update to the relevant `SKILL.md` file.
- Show the proposed change as a diff (before/after the specific lines).
- Ask for approval with `AskUserQuestion` (options: "Apply it" / "Skip").
- If approved: update the skill file, commit with `improve: [skill-name] — [one-line description]`, push, then run `/template-sync` to keep `AGENTS.example.md` in sync.
- Keep changes minimal — fix only what caused the issue, don't refactor.
