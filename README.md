# Product OS

An open source methodology for evidence-guided product management in an AI-native world.
Built for public sector and education product managers, but applicable to any PM working in a mission-driven, resource-constrained organisation.

## The problem this solves

Most PM operating systems are built for VC-backed startups. They assume fast iteration cycles, dedicated research teams, and commercial metrics. Public sector and education PMs work differently — navigating policy alignment, institutional complexity, ethical constraints on experimentation, and users who are hard to reach.

At the same time, most AI-assisted PM tools treat AI as an output accelerator — write PRDs faster, generate user stories quicker. This misses the point. AI's real value in product management is in helping us do genuinely better evidence-guided work — synthesising more evidence, surfacing tensions earlier, maintaining richer understanding of our users and context than was previously practical.

The Product OS is built on that premise.

## What's in here

- **Methodology** — the principles and practices for each component of the OS.
- **Prompts** — reusable, context-agnostic AI prompts you can use with your own Knowledge Core.
- **Templates** — blank structural scaffolding that defines how Knowledge Core files and discovery outputs should be formatted. You populate these with your own organisational context in your private Knowledge Core repo.

## What's not in here

Your data. The OS is a methodology, not a knowledge base. You bring your own organisational context — your strategy, your user research, your signals — structured in a private Knowledge Core repo that references this methodology.

## The four components

| Component | What it does |
|---|---|
| Knowledge Core | Living evidence base — direction, users, signals, beliefs |
| Discovery Engine | Continuous discovery practices, AI-native |
| Decision Layer | Evidence-backed prioritisation and commitment |
| Operational Layer | Automated admin, feeds Knowledge Core as byproduct |

## Getting started

1. Read `AGENTS.md` — agent briefing and design rationale
2. Read `/methodology/knowledge-core/` — build your Knowledge Core first
3. Set up your private Knowledge Core repo using the templates in `/templates/knowledge-core/`
4. Point your Knowledge Core `AGENTS.md` at this repo for methodology and prompts

## Principles

- Evidence over opinion
- Synthesis over storage
- Continuous over periodic
- Portable over bespoke
- Public sector first
